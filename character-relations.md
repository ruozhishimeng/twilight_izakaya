# 《黄昏居酒屋》角色关系文档

## 概述

本游戏角色分为四类：**常人**（仍在世的普通人）、**滞留者**（执念未解的亡灵）、**迷失者**（遗忘自我的游魂）、**鬼神**（神明/精怪）。核心剧情围绕一对兄妹亡灵——阿相与小雪——展开，狐面大叔是贯穿始终的引路人。

> **⚠️ 重要说明**：本文档为剧情开发指导文档，随游戏内容迭代更新。所有关系以 `src/assets/character/*/nodes_main.yaml` 中实际对话内容为准。

## 核心角色（店铺相关）

### 狐面大叔（稻荷）— 鬼神 / 千年调酒师

```
狐面大叔
├── 真实身份：稻荷神（Harvest Deity）
├── 角色：居酒屋初代经营者 → 现任调酒师（玩家）的守护者
├── 外貌变化：千年前是壮年神明 → 如今老迈，拄拐杖，已非永生之身
├── 与玩家（调酒师）：
│   ├── 千年前，玩家是一个被战争摧残的孩子，徘徊于生死边缘
│   ├── 孩子天真、勇敢、聪明、有梦想——但战争碾碎了一切
│   ├── 稻荷本该帮孩子放下执念、平静离开
│   ├── 但他没有——他把自己永恒的生命折了一半，渡进孩子魂魄
│   ├── 代价：孩子活了下来但失去所有记忆，稻荷从此会老会死
│   ├── 玩家替代他成为调酒师，他以"熟客"身份在旁守护
│   ├── 称玩家为「你这小子」——语气熟稔，底下是父亲般的沉默守护
│   ├── 反复教同一杯酒，因为玩家记忆被不断重置
│   ├── 脸上的狐狸面具是那个孩子当年送的——路边十几文，戴了千年
│   └── 从不点破真相——怕说了你仍想不起来，更怕你想起来之后愧疚
└── 与居酒屋：
    ├── 守护店铺千年，识得每一只杯盏
    └── 知晓所有亡灵的名字与执念
```

**核心冲突**：稻荷为玩家放弃了永生。他不是在"赎罪"——他主动选择了那条路，从不后悔。他一遍遍教你，不是愧疚，是守护。他的白头、拐杖、缓慢的步伐，都是爱过一个孩子的证据。

**与玩家的关系**：不是男女之情，是父辈对孩子的守护。稻荷是丰收之神，守护稻田、仓库、屋檐下的灯火——守护"生活"本身。而那个被战争夺走一切的孩子，就是他最想守护却没能守护住的"生活"。他放弃永恒去挽回，不是出于亏欠，是出于一个简单的决定：这个孩子值得活下去。

### 玩家 / 小老板 — 重生者

```
玩家
├── 前世：战争中流离失所的孩子，天真、勇敢、聪明、有梦想
├── 在生死边缘走进居酒屋，遇到正值壮年的稻荷
├── 稻荷放弃永恒生命，将你从消散边缘拉回
├── 今生：替代稻荷成为调酒师，但失去所有前世记忆
├── 记忆不断重置，每一次醒来都是新的开始
├── 狐面大叔的"徒弟"（已被教过无数次同样的配方）
├── 可制作「忘却之酒」帮助亡灵放下执念
├── 似乎是唯一能同时接触滞留者与迷失者的人类
│   —— 因为自己就是从生死边界上被拉回来的
└── 核心谜题：何时能打破记忆循环，认出吧台对面那个为你变老的神
```

**与狐面大叔的关系**：你不是他的徒弟，你是他用半条命换来的孩子。他一遍遍教你，不是因为你是新人，是因为你每一次醒来都会忘记他。你看他白发苍苍、拄着拐杖，以为他生来如此——不知道千年前他站在吧台后面时，还是个调酒手稳得像季节本身的壮年神明。

### 核心物件：红绳（平安绳）

小雪12岁生日时送给哥哥的祈福信物。哥哥把断掉的红绳缝进袖口，不愿让妹妹知道"平安绳"失效了。

> **对话来源**（aqiang_011）：「她12岁的时候系的......她说，这个会保护我。后来绳子磨断了......我没有扔掉，把它缝进了袖口里。我不想让她知道，'平安绳'断了。」

### 核心物件：音乐盒

阿相用167次配送工资购买的生日礼物，内录德彪西《月光》。在小雪18岁生日当天送达途中遭遇车祸，盒子碎裂，阿相的最后一念是「还好，没有碎」。

### 关键关联：黑猫·小墨

小雪饲养的黑猫，在那场雨夜走丢。**阿相的叙述中提到「一只黑猫挡了一下」**——这只猫试图提醒主人危险，却间接导致了阿相的车祸。小墨后来被好心人收养，与小雪失散。

> 这形成了悲剧的完整闭环：猫试图救主人，却让主人错过了最后一面。

### 兄妹线的叙事张力

1. **时空错位**：两人在不同日期到店，从不碰面
2. **认知错位**：小雪不知道哥哥已死；阿相不知道自己已经死了
3. **物件闭环**：红绳、音乐盒、猫——三个物件串联起完整的悲剧
4. **玩家的桥梁角色**：通过调酒和对话，促成「告知」的发生

## 文档迭代指南

### 新增角色流程

1. **在 `src/assets/character/` 下创建角色目录**
2. **编写 `character_meta.yaml`**（角色元数据）
3. **编写 `nodes_main.yaml`**（对话树）
4. **更新本文档**：
   - 在「角色分类」中添加新角色
   - 在「关系矩阵」中添加行/列
   - 在「关系图谱总览」中标注位置
   - 如有新伏笔，在「待解之谜」中添加

### 新增关系流程

当 `nodes_main.yaml` 中出现跨角色引用时：
1. 搜索所有 `nodes_main.yaml` 中的 `「...」` 对话内容
2. 识别提到其他角色的对话节点
3. 在「关系矩阵」中建立双向关系
4. 在「晴山家」或对应角色章节中添加关系说明

### 对话来源标注规范

文档中引用对话时，统一使用以下格式：

```
> **来源**（角色ID_event_id）：「对话原文」
```

### 角色分类标签说明

| 标签 | 含义 | 关键特征 |
|------|------|----------|
| **常人** | 仍在世的普通人 | 有完整人生，有来有去 |
| **滞留者** | 执念未解的亡灵 | 困于死亡时刻，不知自己已死 |
| **迷失者** | 遗忘自我的游魂 | 执念深重，但不知自己是谁 |
| **鬼神** | 神明/精怪 | 超自然存在，见证无数亡魂 |

### 核心悲剧的扩展原则

晴山家悲剧线是游戏的核心叙事骨架。任何新增角色或剧情线，应：

1. **不破坏兄妹线的时间错位设计**（W1_D3 vs W1_D5）
2. **不直接连接兄妹**（玩家是唯一桥梁）
3. **可扩展的方向**：
   - 常人角色与亡灵角色的间接交集（如上班族曾在同一天见过阿相）
   - 狐面大叔与晴山家的历史渊源
   - 玩家身世之谜与晴山家的关联
   - 佐藤奶奶作为小雪重要他人是否可以出现在回忆中
