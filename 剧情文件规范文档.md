# 《黄昏居酒屋》剧情文件规范文档 v1.0

## 1. 目标

本规范用于统一《黄昏居酒屋》的剧情、音频、立绘、奖励、调酒、观察等内容写法，使其满足以下目标：

- 剧情内容与前端逻辑解耦
- 支持 AI 辅助生成与人工编辑协作
- 支持按节点切换 BGM、播放音效
- 支持按节点推进切换对话立绘
- 保证字段缺失时可降级运行，不直接崩溃
- 保证后续角色扩展、支线扩展、演出扩展的可维护性

## 3. 文件职责

### `character_meta.yaml`

负责角色静态资料：

- 姓名、类型、简介
- 立绘资源索引
- 图鉴展示资料
- 默认演出信息

### `nodes_main.yaml`

负责主线剧情节点。

### `nodes_teaching.yaml`

负责教学调酒节点。

### `nodes_side.yaml`

负责支线、特殊事件节点。

### `observations.yaml`

负责观察点与观察结果。

### `gallery.yaml`

负责图鉴拓展资料。

### `diary.yaml`

负责日记补充资料。

## 4. 节点总结构

每个剧情节点建议统一为一个对象：

```yaml
- event_id: fox_uncle_001_dialogue
  type: dialogue
  next_node: fox_uncle_001_reply
  trigger_condition:
  script_flow:
  player_options:
  observation:
  mixing:
  reward:
  diary_note:
  presentation:
  audio:
  flags:
```

## 5. 节点必填字段

所有节点至少应包含：

- `event_id`
- `type`

同时至少满足以下一种内容存在：

- `script_flow`
- `player_options`
- `mixing`
- `reward`
- `observation`

也就是说，一个节点必须有“可执行内容”。

## 6. 节点类型

建议固定以下类型：

- `dialogue`：普通对话节点
- `observation`：进入打量/观察
- `mixing`：进入调酒
- `reward`：奖励展示
- `transition`：黑屏过渡、铃声、日结、独白等
- `system`：纯系统提示
- `ending`：当前客人流程结束

## 7. 文本系统

统一使用 `script_flow` 承载文本：

```yaml
script_flow:
  - type: env
    content:
      - 店门被轻轻推开。
  - type: npc
    content:
      - “今晚还是照旧。”
  - type: inner
    content:
      - 他看起来比昨天更疲惫。
  - type: system
    content:
      - 你记下了这一刻。
```

支持的 `type`：

- `env`：环境描写
- `npc`：角色台词
- `inner`：主角内心独白
- `system`：系统提示

## 8. 分支系统

统一使用 `player_options`：

```yaml
player_options:
  - option: "点头回应"
    next_node: fox_uncle_001_reply_01
    branch_type: flavor
    condition:
    impact_log:
    trigger_event:
```

字段说明：

- `option`：按钮文本
- `next_node`：跳转目标节点
- `branch_type`：可选，`flavor | plot | hidden`
- `condition`：可选，条件解锁
- `impact_log`：可选，日志记录
- `trigger_event`：可选，触发事件标记

## 9. 触发条件

统一使用 `trigger_condition`：

```yaml
trigger_condition:
  need_event:
    - fox_uncle_001_complete
  need_item:
    - m09
  need_time: W1_D3
  need_flag:
    - met_yuki
```

支持：

- `need_event`
- `need_item`
- `need_time`
- `need_flag`

## 10. 音乐与音效系统

所有节点可带 `audio` 字段：

```yaml
audio:
  bgm:
    action: switch
    tag: fox_uncle_theme
    fade_in: 1.2
    fade_out: 0.8
    loop: true
  sfx:
    - tag: door_bell
      timing: on_enter
      volume: 0.9
    - tag: notebook_open
      timing: after_text
      volume: 0.7
```

### `audio.bgm`

用于控制背景音乐。全局同一时刻仅允许一条 BGM 生效。

字段：

- `action`
  - `switch`：切换到新 BGM
  - `stop`：停止当前 BGM
  - `keep`：保持当前 BGM
  - `resume`：恢复上一首
- `tag`：BGM 资源标签
- `fade_in`：淡入时长，单位秒
- `fade_out`：淡出时长，单位秒
- `loop`：是否循环

### `audio.sfx`

用于控制音效。可与 BGM 并行，不冲突。

字段：

- `tag`：音效资源标签
- `timing`
  - `on_enter`
  - `after_text`
  - `on_option_select`
  - `on_transition`
- `volume`：音量，0 到 1

### 音频运行规则

- `bgm.action = keep` 时不切歌
- `bgm.action = switch` 时淡出旧 BGM，再淡入新 BGM
- `sfx` 永远不打断 BGM
- 一个节点可播放多个 `sfx`
- 节点未写 `audio` 时，默认保持当前 BGM

## 11. 立绘与演出系统

所有节点可带 `presentation` 字段：

```yaml
presentation:
  speaker: npc
  portrait:
    action: keep
    character_id: fox_uncle
    expression: serious
    position: center
  background:
    action: keep
    tag: izakaya_night
```

### `presentation.speaker`

只决定对话框显示说话者，不决定立绘是否消失。

建议值：

- `npc`
- `player`
- `system`

### `presentation.portrait`

控制立绘演出。

字段：

- `action`
  - `show`
  - `hide`
  - `keep`
  - `change`
- `character_id`
- `expression`
  - `dialogue`
  - `main`
  - `smile`
  - `serious`
  - `shadow`
- `position`
  - `left`
  - `center`
  - `right`

### 立绘运行规则

- `speaker` 只影响名字牌
- `portrait.action = keep` 时保持当前立绘状态
- `portrait.action = show/change` 时更新立绘
- `system` 文本不强制隐藏立绘
- 角色一旦入场，除非显式 `hide`，否则不自动消失

## 12. 背景切换系统

可在 `presentation.background` 中定义：

```yaml
presentation:
  background:
    action: switch
    tag: mixing_room
    transition: fade
```

字段：

- `action`
  - `switch`
  - `keep`
  - `hide`
- `tag`
- `transition`
  - `fade`
  - `cut`
  - `blackout`

## 13. 调酒节点规范

调酒统一使用 `mixing` 字段：

```yaml
mixing:
  mode: teaching
  speaker: teacher
  teaching_prompt:
    - “先稳住基底。”
    - “最后让味道收回来。”
  player_prompt:
  preferred_drink:
    id: R034
    name: 止痛药
    formula: [bc04, m13, m11]
  retry_on_fail: true
  on_success: fox_uncle_003_success
  on_fail: fox_uncle_003_fail
```

字段说明：

- `mode`
  - `teaching`
  - `normal`
- `speaker`
  - `teacher`
  - `player`
  - `system`
- `teaching_prompt`
  - 只用于教学调酒
  - 对话框显示老师口吻
- `player_prompt`
  - 只用于普通调酒
  - 对话框显示主角总结
- `preferred_drink`
  - 必须包含 `id` 和 `formula`
- `retry_on_fail`
- `on_success`
- `on_fail`

## 14. 奖励节点规范

奖励统一列表化：

```yaml
reward:
  items:
    - type: ingredient
      id: m09
      name: 乌龙茶
      is_new: true
    - type: recipe
      id: R041
      name: 流金岁月
      is_new: true
```

字段：

- `type`
  - `ingredient`
  - `recipe`
  - `memory`
  - `key_item`
- `id`
- `name`
- `is_new`

## 15. 日记与反思规范

节点结束后可记录主角感受：

```yaml
diary_note:
  text:
    - 今晚他说的话像酒一样发苦。
    - 但他似乎并不需要安慰。
```

规则：

- `diary_note.text` 建议为数组
- 运行时可拼接为黑屏独白
- 同时写入日记系统

- `prompt`
- `feature_groups`
- `continue_node`

## 18. AI 生成内容的校验要求

建议将字段分为两类。

### 关键必填

- `event_id`
- `type`
- 对话节点的 `script_flow`
- 分支节点的 `player_options` 或 `next_node`
- 调酒节点的 `mixing.preferred_drink`
- 奖励节点的 `reward.items`

### 可选字段

- `audio`
- `presentation`
- `diary_note`
- `trigger_condition`
- `impact_log`

## 19. 缺字段时的运行时策略

为了避免系统崩溃，运行时建议：

- 缺 `audio`：默认保持当前 BGM，不播放 SFX
- 缺 `presentation`：默认 `portrait.keep`
- 缺 `speaker`：按文本类型推导
- 缺 `reward.items`：跳过奖励展示
- 缺 `diary_note`：跳过反思记录
- 缺 `next_node`：视为当前分支结束
- 缺关键 `event_id`：直接判为非法节点，不载入

## 20. 完整节点示例

```yaml
- event_id: fox_uncle_003_dialogue
  type: dialogue
  next_node: fox_uncle_003_mixing
  script_flow:
    - type: env
      content:
        - 门铃轻轻响了一声。
    - type: npc
      content:
        - “今天，教你一杯不该轻易用上的酒。”
    - type: inner
      content:
        - 他的语气比平时更低。
  audio:
    bgm:
      action: switch
      tag: fox_uncle_memory
      fade_in: 1.5
      fade_out: 1.0
      loop: true
    sfx:
      - tag: door_bell
        timing: on_enter
        volume: 0.8
  presentation:
    speaker: npc
    portrait:
      action: change
      character_id: fox_uncle
      expression: serious
      position: center
    background:
      action: keep
      tag: izakaya_night

- event_id: fox_uncle_003_mixing
  type: mixing
  mixing:
    mode: teaching
    speaker: teacher
    teaching_prompt:
      - “先让基底稳下来。”
      - “最后用一点果味把苦压回去。”
    preferred_drink:
      id: R034
      name: 止痛药
      formula: [bc04, m13, m11]
    retry_on_fail: true
    on_success: fox_uncle_003_reward
    on_fail: fox_uncle_003_fail
  audio:
    bgm:
      action: switch
      tag: mixing_teaching
      fade_in: 0.8
      fade_out: 0.5
      loop: true

- event_id: fox_uncle_003_reward
  type: reward
  next_node: fox_uncle_003_end
  reward:
    items:
      - type: ingredient
        id: m13
        name: 椰浆
        is_new: true
  audio:
    sfx:
      - tag: reward_chime
        timing: on_enter
        volume: 0.9
  presentation:
    speaker: system
    portrait:
      action: keep
  diary_note:
    text:
      - 今晚学到的，不只是配方。
```

## 21. 最终建议

后续内容生产统一遵循：

- 所有剧情行为都通过节点字段显式声明
- 所有音频行为都写入 `audio`
- 所有立绘切换都写入 `presentation`
- 不再依赖前端“猜测当前节点应该怎么表现”
- 先以 `fox_uncle` 为样板角色做完整规范，再迁移其他角色

## 22. 下一步建议

建议后续继续补两份文档：

1. `character_meta.yaml` 模板
2. `nodes_main.yaml / nodes_teaching.yaml` 模板

这样后续无论是人工编写还是 AI 生成，都能统一按模板产出。
