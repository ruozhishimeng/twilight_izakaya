# 《黄昏居酒屋》状态系统驱动节点模板

更新时间：2026-04-17  
文档定位：内容作者最小可用模板  
优先级：字段契约以 `状态系统驱动内容规范.md` 为准，本文件只提供可直接复制的骨架

## 1. 使用原则

这份模板只服务一件事：

- 让内容作者按当前运行时真实支持的字段写内容

不再使用以下思路：

- 先写旧字段，等前端去猜
- 缺字段靠运行时兜底
- 用节点 `type` 猜阶段

写完任何内容后，都必须执行：

```powershell
Set-Location 'F:\twilight_izakaya'
npm run content:check
```

---

## 2. 最小工作流

### 2.1 新增或修改主线节点

1. 改 `nodes_main.yaml`
2. 如果涉及观察，补 `observations.yaml`
3. 如果涉及故事解锁，补 `gallery.yaml`
4. 跑 `npm run content:check`

### 2.2 新增或修改教学节点

1. 改 `nodes_teaching.yaml`
2. 确认 `recipe.id` / `formula` 在 `recipes.json` 中真实存在
3. 确认成功/失败出口真实存在
4. 跑 `npm run content:check`

### 2.3 新增或修改闲聊/隐藏节点

1. 改 `nodes_chat.yaml` 或 `nodes_hidden.yaml`
2. 必须使用 `atmosphere_lines + content`
3. 不要再写旧 `script_flow` 聊天格式
4. 跑 `npm run content:check`

## 3. 主线节点模板

适用文件：

- `nodes_main.yaml`

```yaml
character_id: example_guest
nodes:
  - event_id: example_001_dialogue_main
    trigger_condition:
      need_event: []
      need_time: any
      need_item: []
    script_flow:
      - type: env
        content:
          - 店门被轻轻推开。
          - 雨水顺着伞骨滴在门槛边。
      - type: npc
        content:
          - "「今晚......还有位置吗？」"
      - type: inner
        content:
          - 他像是刚从很远的地方赶来。
    player_options:
      - text: "「请进。」"
        branch_type: flavor
        script_flow:
          - type: npc
            content:
              - "「......谢谢。」"
        impact_log: 我请他进门坐下
      - text: "「你看起来淋湿了。」"
        branch_type: flavor
        script_flow:
          - type: npc
            content:
              - "「外面雨大，习惯了。」"
        impact_log: 我注意到他浑身湿透
    next_node: example_002_dialogue_followup
```

适用规则：

- `main` 节点必须写 `script_flow`
- `script_flow.type` 只能是 `env / npc / inner`
- `player_options` 没有也可以

## 5. 点单节点模板

适用文件：

- `nodes_main.yaml`

```yaml
  - event_id: example_003_drink_request
    trigger_condition:
      need_event:
        - example_002_after_observation
      need_time: any
      need_item: []
    script_flow:
      - type: env
        content:
          - 他看着空酒杯，沉默了一会儿。
      - type: npc
        content:
          - "「给我一杯......能让我冷静下来的酒。」"
    player_options: []
    next_node: null
    drink_request:
      request_text: 给我一杯能让我冷静下来的酒
      hint: 他说话时手指一直在抖，像在强压情绪
      retry_on_fail: false
      preferred_drink:
        id: "R021"
        name: "纯米嗨棒"
        formula: ["bj01", "m01"]
    on_mixing_fail: example_phase1_regular_success
```

正式要求：

- `request_text` 必填
- `preferred_drink.id / name / formula` 必填
- 至少有一个出口：
  - `on_mixing_complete`
  - `on_mixing_fail`
  - 或存在依赖该节点的结果节点

## 9. 图鉴章节模板

适用文件：

- `gallery.yaml`

```yaml
character_id: example_guest
base_info:
  name: 例子客人
  type: 迷失者
  description: 总在下雨前后出现的沉默来客。

profile_sections:
  - id: surface
    title: 雨夜来客
    content: |
      他总在门铃最轻的时候进门，像是不想惊动任何人。
      外套总是半湿，鞋边沾着洗不掉的泥。
    unlock_condition: example_phase1_arrival
    depth_level: 1

  - id: memory_after_rain
    title: 雨停之后
    content: |
      他终于承认，那场雨里真正没能回来的人不是别人，而是过去的自己。
    unlock_condition: example_phase3_resolution_end
    depth_level: 3
```

正式要求：

- 每个章节都必须有 `id`
- 每个章节都必须有 `title`
- 每个章节都必须有 `content`
- 每个章节都必须有 `depth_level`
